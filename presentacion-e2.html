Entregable 2

Objetivo de la Entrega

El presente entregable constituye el Producto Mínimo Viable (MVP) operativo para la empresa TA-DIN. Su propósito es validar de extremo a extremo (End-to-End) la arquitectura técnica implementada, garantizando la correcta comunicación entre la interfaz de usuario, la lógica de negocio en el servidor y la persistencia de datos en el motor relacional, sentando las bases para la futura certificación de procesos bajo la norma ISO 9001.

Requisitos por Capas del Desarrollo

A. Página Web
Frontend — Interfaz y Navegación
  • Vistas Clave: Home y Dashboard completos, basados en el mockup.
  • Navegación Fluida: Enrutamiento real entre vistas sin recargas innecesarias.
Backend y Base de Datos
  • Base de Datos Activa: Tablas críticas con restricciones y tipos de datos correctos.
  • API Operativa: Al menos 2 endpoints HTTP funcionales (listado y envío/filtrado de datos).
B. Reloj Biométrico
1. Interfaz de Empleado
  • Pantalla de espera: Reloj digital en tiempo real + instrucción de huella.
  • Confirmación: 2 s con foto, nombre, hora e indicador Verde = Éxito / Rojo = Error.
2. Panel de Administrador
  • Dashboard: Gráficos en tiempo real (presentes, ausentes, tardanzas).
  • Gestión: Registro de empleados y enrolamiento de huellas.
  • Reportes: Filtros y exportación del historial de asistencia.
Backend — ZKBioTime.Net
  • Motor BD: SQLite (default) · compatible SQL Server / MySQL / PostgreSQL.
  • Tabla Usuarios: ID_Usuario, Nombre, Apellido, plantillas biométricas.
  • Tabla Registros: ID_Registro, ID_Usuario, FechaHora_Marcacion, Tipo_Movimiento.
  • Protocolo: TCP/IP directo · Puerto 4370 (no REST/HTTP estándar).

1

Interfaz del Sistema

El software ZKBioTime.Net actúa como middleware oficial del dispositivo biométrico, gestionando de forma automática la estructura relacional de los datos.

Se instala con una base de datos local SQLite por defecto, aunque su arquitectura permite la conexión directa con motores más robustos como SQL Server, MySQL o PostgreSQL.

La comunicación entre el reloj y el software se realiza directamente por protocolo TCP/IP en el Puerto 4370, sin necesidad de endpoints HTTP convencionales.
